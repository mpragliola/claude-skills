# My Claude Skills

Some Claude Code skills I wrote for my personal projects.

## Unfurl

The Skill you officially will never use. Did you falter in your GIT discipline?
Is there a mess in your working tree? Unfurl helps making some order by analyzing
the changes and commit them in a way that is atomic and makes sense chronologically,
including committing partial files (hunks) when necessary.

> **Warning**: this skill was created exclusively as a helper to untangle things
> in small pet projects, and only in the cases where it's okay to relax the quality
> constraints. It should never substitute proper GIT and Version Control discipline
> and never be used in any serious development process.

## Nigel

Nigel is a mock IT interviewer persona. It focuses on high-level topics (senior or
principal level) and allows to practice mock interviews about coding concept, system
design, or any IT topic the user prefers. It's especially thought for **people who
take advantage of mock interviews as a learning device**.
Nigel is:
- direct and pragmatic, tries to **avoid fillers and LLM sycophantisms** and flags
  wrong or handwavy answers.
- can be instructed to give a **short**, **standard** or **extended interview**
- when your answer is not good and polished enough, Nigel uses the **Socratic method**.
  He nudges you slightly towards the correct anser rather than serving the solution on
  a plate, and forces you to think about the issue and get to it step by step.
- when stuck, or when a deeper explanation is needed, a **teacher mode** can be activated
  where Nigel breaks out of character and becomes a helpful assistant.
- can be used also for **soft skill topics**.
- offers a **feedback** at the end of the round
